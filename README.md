# <p align="center"> Проект интернет-магазина </p>
Заготовка для интернет магазина с использованием фреймворка django

##  <p align="center">Установка:</p>

1. Клонируйте репозиторий: git@github.com:gabdrakhmanov-D//An-example-of-an-online-store.git
2. Обновите зависимости с помощью команды `poetry update`.

## <p align="center">Описание:  </p>

Проект содержит два приложения: catalog и blog. В приложении catalog используются такие веб-страницы как:
  - Главная.
  - Контакты с формой обратной связи.
  - Детали о товаре.
  - Страница добавления товара в каталог.
    
В приложении blog  используются страницы:
  - Главная со списокм статей.
  - Страница добавления статьи.
  - Страница редактирования статьи.
  - Страница удаления статьи
    
В обоих приложениях используются шаблоны и подшаблоны.

## <p align="center">Запуск:</p>

Для демонстрации работы приложения, активируйте виртуальное окружение и выполните команду `python manage.py runserver`.

Чтобы подключить базу данных заполните шаблон .env.sample

## <p align="center">Особенности:</p>

В приложении есть форма обратной связи, все данные полученные из этой формы выводятся в консоль.

## <p align="center">Изменения в версии 2.0:  </p>
- Подключена база данных PostgreSQL. Настройки работы с БД вынесены в переменные окружения в файл .env.
- Созданы модели товаров, категорий и контактов магазина. Выполнены миграции.
- Для этих моделей настроено корректное отображение в административной панели.
- Сформированы фикстуры для продуктов и категорий.
- Реализована кастомная команда для загрузки объектов.

## <p align="center">Изменения в версии 3.0:  </p>
- Реализована страница одного товара.
- Создан новый контроллер для одного товара.
- В контроллер главной страницы добавлен ORM-запрос на получение списка товаров.
- С главной страницы можно перейти на страницу товара и назад.
- Выделен базовый шаблон.
- Базовый шаблон используется в шаблонах главной страницы, контактов и одного товара.
- Создан подшаблон с главным меню.

## <p align="center">Изменения в версии 4.0:  </p>
- Все контроллеры приложения catalog переведены в CBV.
- Создано нове приложение блога, в нем реализованы следующие функции:
    - При открытии отдельной статьи увеличивается счетчик просмотров статьи.
    - На страницу списка статей выводятся только статьи, которые имеют положительный признак публикации.
    - После успешного редактирования записи происходит перенаправление пользователя на страницу отредактированной статьи.
